Elige la plataforma de despliegue
✓ Por qué RailwayFree tier de 500 horas/mes, sin spin-down automático, deploys en segundos desde GitHub, soporte nativo para Procfile.
1

Crea el repositorio en GitHub

Railway deploya desde GitHub. Si aún no tienes el repo:

cd bia_trading_copilot
git init
git add .
git commit -m "initial: BIA Trading Copilot"
# Crea el repo en github.com y luego:
git remote add origin https://github.com/TU_ORG/bia-trading-copilot.git
git push -u origin main
2

Crea el proyecto en Railway

  1. Ve a railway.app → New Project
  2. Elige Deploy from GitHub repo
  3. Selecciona bia-trading-copilot
  4. Railway detecta el Procfile automáticamente
3

Agrega las variables de entorno

En el panel de Railway: Variables → Raw Editor → pega esto y rellena los valores:

SUPABASE_URL=https://zzxyqziekjylvgumvpft.supabase.co
SUPABASE_KEY=<tu_service_role_key>
ANTHROPIC_API_KEY=<tu_anthropic_key>
SLACK_BOT_TOKEN=<xoxb-tu-token>
SLACK_CHANNEL_ID=C0B5K5KPEGP
ℹ PORTRailway inyecta $PORT automáticamente. El Procfile ya lo usa — no agregues esta variable.
4

Obtén la URL pública

En Railway: Settings → Networking → Generate Domain. La URL será algo como:

https://bia-trading-copilot.up.railway.app

Verifica que el servidor responde:

curl https://bia-trading-copilot.up.railway.app/health
# → {"status": "ok"}
5

Conecta con Slack

En api.slack.com/apps → tu app → Event Subscriptions:

  1. Activa Enable Events
  2. Request URL: https://bia-trading-copilot.up.railway.app/slack/events
  3. Slack enviará el challenge → el servidor responde → ✓ Verified
  4. En Subscribe to bot events agrega: app_mention
  5. Guarda y reinstala la app en tu workspace
⚠ Spin-down en free tierRender apaga el servicio tras 15 min de inactividad. El primer request tarda ~30s en reiniciarlo. Usa Railway si necesitas respuesta inmediata.

Correr la ingesta

El script ingest.py es independiente del servidor. Puedes correrlo localmente o programarlo:

# Local (con .env presente)
python ingest.py

# En Railway como Cron Job (panel → New → Cron Job)
# Comando: python ingest.py
# Schedule: 0 * * * *   ← cada hora
💡 Cron en RailwayRailway permite agregar un servicio de tipo Cron Job en el mismo proyecto, usando la misma imagen. Es la forma más limpia de automatizar la ingesta sin servidor adicional.
Checklist de verificación
  • Servidor responde — GET /health retorna {"status":"ok"}
  • Challenge de Slack verificado — URL marcada como Verified en api.slack.com
  • Bot invitado al canal — /invite @NombreDelBot en el canal
  • Mention funciona — escribe @bot ¿qué simulaciones hay? y responde en <5s
  • Ingesta ejecutada — simulation_results tiene filas en Supabase
  • .env NO está en el repo — git ls-files .env no retorna nada
